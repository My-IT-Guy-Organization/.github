# Contributing to My IT Guy Repositories

## Development Workflow

1.  Fork the repository or create a feature branch off main (e.g., \feature/issue-number-description`).`

2.  Ensure local testing is completed.

3.  Commit changes using descriptive commit messages.

4.  Push to your branch and open a Pull Request.

## Code Standards

*   Ensure scripts are thoroughly documented.

*   Do not commit sensitive API keys or credentials. Use GitHub Secrets.

*   Verify compatibility with our standardized hardware and environments.
